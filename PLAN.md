# OpenClaw × Threads 자동화 프로젝트 플랜

> **목표**: OpenClaw를 활용해 퀄리티 있는 Threads 콘텐츠를 자동 생성·스케줄링·게시·분석하는 시스템 구축
> **작성일**: 2026-04-05

---

## 1. 리서치 요약

### 1.1 OpenClaw 개요

| 항목 | 내용 |
|------|------|
| **정체** | MIT 라이선스 오픈소스 AI 에이전트 프레임워크 (GitHub ⭐ 347K+) |
| **언어** | TypeScript |
| **핵심 아키텍처** | Gateway (데몬) → Agent Runtime (LLM + Tools + Skills) → 외부 API |
| **비용** | OpenClaw 자체 무료. LLM API 사용료만 발생 (Anthropic/OpenAI 등) |
| **주요 기능** | 크론 스케줄러, Skills 시스템, 웹훅 API, 멀티채널 연동 (Telegram/Discord 등) |
| **SOUL.md** | 브랜드 보이스·톤·규칙을 정의하는 파일. 모든 세션에서 참조됨 |

### 1.2 Threads API (Meta Graph API)

| 항목 | 내용 |
|------|------|
| **엔드포인트** | `https://graph.threads.net/v1.0/` |
| **게시 방식** | 2단계 컨테이너 모델: ①컨테이너 생성 → ②게시 |
| **지원 포맷** | TEXT, IMAGE, VIDEO, CAROUSEL (최대 20장) |
| **추가 기능** | 토픽 태그 (1개/게시물), 링크 첨부, GIF 첨부 (Tenor) |
| **레이트 리밋** | 250 게시물/24시간 (답글은 미포함) |
| **필요 권한** | `threads_basic`, `threads_content_publish`, `threads_manage_insights` |
| **앱 리뷰** | 필수 (2~4주 소요). 스크린캐스트 데모 제출 필요 |
| **토큰 수명** | 장기 토큰 60일 → 자동 갱신 로직 필요 (만료 시 재인증 필수) |
| **토큰 교환 URL** | `https://graph.threads.net/access_token` (Facebook Graph API가 아닌 별도 URL) |
| **스케줄링** | 네이티브 스케줄링 없음 → 자체 큐+크론 구현 필요 |
| **추천 Python 라이브러리** | `pythreads` (★70, 비동기, 93% 테스트 커버리지, 가장 완성도 높음) |

### 1.3 사용 가능한 OpenClaw Threads 스킬

| 스킬 | 방식 | 설명 |
|------|------|------|
| **`threads`** (ruslanlanket) | Graph API 직접 호출 | Python CLI. 텍스트/이미지/캐러셀 게시, 프로필 조회, 포스트 리스트 |
| **`spool`** (zizi-cat) | 브라우저 자동화 | OpenClaw 브라우저 툴로 Threads 웹사이트 직접 조작. API 없이도 동작 |
| **`social-media-autopilot`** (reighlan) | 크로스 플랫폼 | 콘텐츠 캘린더, 승인 워크플로, 분석 리포트 통합 관리 |
| **`instagram-api`** (lifeissea) | Graph API + Imgur | Threads/Instagram 직접 게시. 미디어 호스팅 포함 |

### 1.4 Threads 알고리즘 & 콘텐츠 전략 핵심

#### 알고리즘 작동 원리
- **For You 피드**: AI 랭킹 (메인 도달 채널)
- **Following 피드**: 순수 시간순 (알고리즘 없음)
- 3단계 프로세스: 콘텐츠 인벤토리 수집 → 시그널 분석 → 랭킹

#### 랭킹 팩터 (중요도 순)

| 팩터 | 가중치 | 설명 |
|------|--------|------|
| **인게이지먼트 속도** | 매우 높음 | 30분 내 50좋아요 > 24시간에 100좋아요 |
| **답글 깊이** | 매우 높음 | 주고받는 대화가 단순 좋아요보다 중요 |
| **좋아요/리포스트/공유** | 높음 | 기본 인게이지먼트 시그널 |
| **포스트 체류 시간** | 높음 | 읽기 완료 = 깊은 관심 |
| **관계 강도** | 높음 | 특정 작성자와의 상호작용 이력 |
| **콘텐츠 관심 매칭** | 높음 | 이전 인게이지먼트 패턴과 일치하는 주제 |
| **최신성** | 높음 | 새 콘텐츠 부스트 |

#### 콘텐츠 포맷 성과 (Buffer 4,500만+ 포스트 분석, 2026.03)

| 포맷 | 중앙값 인게이지먼트율 | 비고 |
|------|----------------------|------|
| **비디오** | 5.55% | 🥇 최고 성과 — 텍스트의 거의 2배 |
| **이미지** | 4.55% | 🥈 텍스트 대비 +60% |
| **텍스트** | 2.79% | 기본 포맷, 500자 제한 |
| **링크** | 2.34% | 🚫 최저 — 본문에 링크 넣지 말 것 |

#### 핵심 전략 포인트
- **답글 > 게시**: Adam Mosseri "답글을 게시보다 더 많이 하라. 답글의 총 가치 ≈ 게시물의 총 가치"
- **답글 인게이지먼트 부스트**: 자신의 댓글에 답글하면 +42% 인게이지먼트 (Buffer, 12.8만 포스트 분석 — 전 플랫폼 최고 수치)
- **답글은 레이트 리밋에 포함 안 됨**: 250개/일 한도는 원 게시물만 카운트
- **첫 1시간이 결정적**: 초반 인게이지먼트 속도가 배포 범위 결정
- **토픽 태그**: 게시물당 1개만 가능, 태그 있는 게시물이 조회수 더 높음
- **인게이지먼트 미끼 금지**: "좋아요 누르기" 등은 알고리즘이 강등
- **최적 빈도**: 주 3~5회 (공격적 성장: 1일 1~3회)
- **최적 시간**: 목요일 오전 9시 (최고), 수요일 정오, 평일 오전 9~11시
- **최악 시간**: 저녁 6~11시 (모든 요일에서 저조)
- **500자 제한**, 이모지는 UTF-8 바이트로 계산
- **링크는 첫 답글에**: 본문 링크 → 2.34% vs 비디오 → 5.55%. 링크는 첫 답글에 넣기

## 2. 시스템 아키텍처

```
┌──────────────────────────────────────────────────────────────┐
│                    OpenClaw Gateway (자체 호스팅)               │
│                                                              │
│  ┌─────────────┐    ┌──────────────┐    ┌────────────────┐  │
│  │  SOUL.md    │    │  콘텐츠 파이프│    │   분석 엔진    │  │
│  │ (브랜드 보이스) │    │  라인       │    │               │  │
│  └─────────────┘    └──────────────┘    └────────────────┘  │
│                                                              │
│  크론 스케줄러:                                                │
│  ├─ 07:00 트렌드 리서치 (web-search 스킬)                      │
│  ├─ 08:00 콘텐츠 생성 & 리뷰 요청                              │
│  ├─ 09:00/12:00/15:00 승인된 콘텐츠 게시                       │
│  ├─ 18:00 인게이지먼트 확인 & 답글 초안                         │
│  └─ 금요일 17:00 주간 분석 리포트                               │
│                                                              │
│  스킬 스택:                                                    │
│  ├─ threads (Graph API 직접 게시)                              │
│  ├─ web-search (트렌드 조사)                                   │
│  ├─ image-generation (비주얼 콘텐츠)                            │
│  └─ social-media-autopilot (캘린더/분석)                       │
└───────────────────────┬──────────────────────────────────────┘
                        │
              Meta Graph API (graph.threads.net)
                        │
                  Threads 계정
```

### 2.1 데이터 플로우

```
[트렌드 수집] → [콘텐츠 생성] → [사람 리뷰/승인] → [최적 시간 게시] → [인게이지먼트 모니터링] → [피드백 루프]
```

1. **트렌드 수집**: web-search 스킬로 업계 트렌드, 경쟁자 동향, 인기 토픽 스캔
2. **콘텐츠 생성**: LLM이 SOUL.md 기반으로 포스트 초안 작성 (이미지 생성 포함)
3. **리뷰/승인**: Telegram/Discord 채널로 초안 전송 → 사용자 승인/수정
4. **게시**: 승인된 콘텐츠를 threads 스킬로 최적 시간에 자동 게시
5. **모니터링**: 인게이지먼트 추적, 답글 초안 제안
6. **피드백 루프**: 주간 분석으로 콘텐츠 전략 개선

### Phase 2: 브랜드 보이스 & 콘텐츠 전략 정의 (반나절)

#### 2.1 SOUL.md 작성

```markdown
# SOUL.md — Threads 브랜드 보이스

## 기본 정보
- 플랫폼: Threads (@your_handle)
- 언어: 한국어 (영어 혼용 가능)
- 톤: 대화체, 직관적, 약간의 위트. 과하게 격식 차리지 않음.

## 콘텐츠 필러 (Content Pillars)
1. [주제 1] — 예: AI/자동화 팁과 인사이트
2. [주제 2] — 예: 개발자 라이프 & 생산성
3. [주제 3] — 예: 업계 뉴스에 대한 의견
4. [주제 4] — 예: 비하인드 스토리 / 개인 경험

## 작성 규칙
- Threads: 최대 500자, 한 문장 훅으로 시작
- 이모지: 게시물당 1~2개, 과하지 않게
- 토픽 태그: 반드시 1개 포함 (게시물 내용과 가장 관련 있는 것)
- 절대 쓰지 않는 표현: "레버리지", "시너지", "게임체인저", "혁신적"
- 첫 줄에 훅: 숫자, 질문, 또는 과감한 주장
- 이미지 동반: 가능하면 항상 이미지 포함 (텍스트 대비 +60% 인게이지먼트)

## 인게이지먼트 규칙
- 답글은 게시만큼 중요 → 매일 5~10개 대화에 참여
- 다른 계정의 포스트에 가치 있는 답글 달기
- 인게이지먼트 미끼 ("좋아요 눌러주세요") 절대 금지
- 부정적이거나 논쟁적인 콘텐츠 피하기

## 포스트 유형
1. 핫테이크 (의견/관점) — 대화 유도
2. 질문형 — 답글 유도  
3. 팁/방법론 — 가치 제공
4. 개인 이야기 — 진정성
5. 트렌드 코멘터리 — 시의성
```

#### 2.2 콘텐츠 캘린더 템플릿

| 요일 | 시간 | 콘텐츠 유형 | 비고 |
|------|------|------------|------|
| 화 | 10:00 | 핫테이크 / 의견 | 주 초반 에너지 |
| 수 | 12:00 | 이미지 + 팁 | 수요일 피크 시간 |
| 목 | 09:00 | 비디오 + 트렌드 코멘터리 | ⭐ 전체 최고 인게이지먼트 슬롯 |
| 금 | 10:00 | 질문형 / 토론 | 주말 전 대화 유도 |
| 금 | 12:00 | 개인 이야기 / 비하인드 | 진정성 콘텐츠 |

### Phase 3: 자동화 파이프라인 구축 (1~2일)

#### 3.1 트렌드 리서치 크론

```bash
openclaw cron add trend-research \
  --cron "0 7 * * 1-5" \
  --tz "Asia/Seoul" \
  --session isolated \
  --message "오늘의 Threads 콘텐츠를 위해 다음을 조사해줘:

1. 내 콘텐츠 필러 관련 최신 트렌드 (SOUL.md 참조)
2. Threads에서 현재 인기 있는 토픽 태그
3. 경쟁 계정 최근 포스트 중 인게이지먼트 높은 것

결과를 간결한 브리핑으로 정리해줘." \
  --channel telegram
```

#### 3.2 콘텐츠 생성 크론

```bash
openclaw cron add content-generation \
  --cron "0 8 * * 1,3,5" \
  --tz "Asia/Seoul" \
  --session isolated \
  --message "오늘 Threads에 올릴 포스트를 2개 생성해줘.

참조:
- SOUL.md의 브랜드 보이스 규칙
- 오늘의 트렌드 리서치 결과
- 콘텐츠 캘린더 (오늘에 해당하는 유형)

각 포스트에 대해:
1. 게시물 텍스트 (500자 이내)
2. 추천 토픽 태그 1개
3. 이미지 필요 여부 & 이미지 프롬프트 (필요시)
4. 추천 게시 시간

전부 리뷰용으로 보여주고, 승인 받으면 게시해." \
  --channel telegram
```

#### 3.3 게시 크론 (승인 후 자동 게시)

```bash
openclaw cron add auto-publish \
  --cron "0 9,12,15 * * 1-5" \
  --tz "Asia/Seoul" \
  --session isolated \
  --message "승인된 콘텐츠 큐를 확인하고, 현재 시간에 예정된 포스트가 있으면 Threads에 게시해줘.

게시 후:
1. 게시 성공 여부 확인
2. 포스트 URL 알려줘
3. 30분 후 초기 인게이지먼트 체크 알람 설정" \
  --channel telegram
```

#### 3.4 인게이지먼트 모니터링 크론


#### 3.5 주간 분석 리포트 크론

```bash
openclaw cron add weekly-analytics \
  --cron "0 17 * * 5" \
  --tz "Asia/Seoul" \
  --session isolated \
  --message "이번 주 Threads 분석 리포트 생성:

1. 총 노출수, 인게이지먼트율, 팔로워 변화
```

### Phase 4: 토큰 자동 갱신 (선택, 고급)

Threads 액세스 토큰은 60일 만료. 자동 갱신 스크립트:

```bash
# ~/.openclaw/scripts/refresh-threads-token.sh
#!/bin/bash
CURRENT_TOKEN=$(cat ~/.openclaw/.env | grep THREADS_ACCESS_TOKEN | cut -d"'" -f2)

NEW_TOKEN=$(curl -s "https://graph.threads.net/refresh_access_token?grant_type=th_refresh_token&access_token=$CURRENT_TOKEN" | jq -r '.access_token')

if [ "$NEW_TOKEN" != "null" ] && [ -n "$NEW_TOKEN" ]; then
  sed -i '' "s|THREADS_ACCESS_TOKEN='.*'|THREADS_ACCESS_TOKEN='$NEW_TOKEN'|" ~/.openclaw/.env
  echo "Token refreshed successfully: $(date)"
else
  echo "ERROR: Token refresh failed: $(date)"
  # Telegram 알림 보내기
fi
```

```bash
# 45일마다 자동 실행
openclaw cron add token-refresh \
  --every 45d \
  --tz "Asia/Seoul" \
  --session isolated \
  --message "Threads 액세스 토큰 갱신 스크립트를 실행해줘: bash ~/.openclaw/scripts/refresh-threads-token.sh"
```

## 4. 콘텐츠 품질 가이드라인

### 4.1 "퀄리티" 정의

Threads 알고리즘이 보상하는 품질:
- **대화 유발**: 답글을 부르는 콘텐츠 (질문, 의견, 토론)
- **구체적 데이터/경험**: "생산성이 올랐다" ❌ → "이메일 처리 시간이 2.5시간에서 15분으로 줄었다" ✅
- **진정성**: 기업체 보도자료 느낌 ❌ → 실제 경험과 개성 ✅
- **시의성**: 현재 트렌드/뉴스에 연결된 콘텐츠
- **비주얼**: 텍스트에 이미지 추가 시 +60% 인게이지먼트

### 4.2 피해야 할 것

| ❌ 하지 말 것 | ✅ 대신 할 것 |
|--------------|--------------|
| "좋아요 누르면 OO" (인게이지먼트 미끼) | 가치 있는 의견으로 자연스러운 참여 유도 |
| 하루 3개 초과 게시 | 주 3~5개, 최대 일 3개 |
| 인스타그램 미학만 추구 | 대화와 개성 중심 |
| 제네릭한 동기부여 문구 | 구체적 데이터, 개인 경험 |
| 다른 콘텐츠 복사/재게시 | 오리지널 관점 추가 |
| 외부 링크를 본문에 넣기 | 첫 번째 답글에 링크 넣기 (선택) |

### 4.3 AI 생성 콘텐츠 품질 관리

```
AI 초안 생성 → 사람 리뷰 → 개인 터치 추가 → 승인 → 게시
```

- AI는 80%의 기계적 작업 처리 (리서치, 초안, 스케줄링)
- **사람이 추가하는 20%**: 개인 일화, 고유한 관점, 감정적 터치
- 모든 게시물은 반드시 사람이 리뷰 후 승인 (초기 단계)
- 자동 승인(`auto_approve: true`)은 충분한 학습 데이터 축적 후에만
